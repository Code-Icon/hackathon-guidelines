## 1. Repository Usage

- Each team will receive one private GitHub repository.
- All development must happen only inside the assigned repository.
- Work done outside the assigned repository will not be considered.

---

## 2. Collaboration

- Team leaders are responsible for managing their team members.
- Only assigned collaborators are allowed to access the repository.
- Sharing repository access with external users is strictly prohibited.

---

## 3. Code Integrity

- Force pushing is not allowed.
- Rewriting commit history is not allowed.
- Deleting commit history to hide development timeline is prohibited.

---

## 4. Fair Usage

- Use of external open-source libraries is allowed unless stated otherwise.
- Plagiarism or copying another team's work will result in disqualification.
- All code must be original or properly credited.

---

## 5. Organizer Rights

The organizing committee reserves the right to:

- Archive repositories at the deadline
- Review commit history
- Disqualify teams violating rules
